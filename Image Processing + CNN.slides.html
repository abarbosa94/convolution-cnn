Quando trabalhamos com processamento de imagens, existem diferentes algoritmos que podemos utilizar dependendo do objetivo que queremos atingir: tirar o rudo de uma imagem, detectar suas bordas, entre outras coisas.

Dentro da rea de processamento de imagem, uma das tcnicas mais difundidas para solucionar esses problemas a Convoluo.

A convoluo tem raizes na matemtica, e pode ser contnua (que usada em processamento de sinais, por exemplo) ou discreta (que usada em processamento de imagens).

Antes de tudo, uma coisa que podemos perceber que, quando o computador "enxerga" uma imagem, ele no v nada alm de nmeros

Mas como calculamos os valores dos pixels das bordas da imagem? Existem basicamente duas opes:

  • Criar pixels "artificiais" ao redor da imagem Esses pixels extras podem ser escolhidos arbitrariamente ou podem ser estimados a partir dos pixels existentes nas suas vizinhanas (como mdia, valor mximo, valor mnimo etc.). Perceba que a janela sempre ocorre de uma forma "centralizada"

A Convoluo, ento, envolve a aplicao de um kernel sobre uma imagem, gerando uma nova imagem como resultado. O kernel tambm uma matriz e, na prtica, ele "desliza" sobre toda a imagem, modificando-a. Para um dado deslocamento, os elementos do kernel so multiplicados pelos pixels da imagem, que, por fim, so somados e resultam em um elemento da imagem convolucionada

  • Ignorar as bordas

De Convolução à Redes Neurais: as "famosas" CNN¶

Vamos imaginar que ns queremos identificar os rostos de uma pessoa em uma rede social, diferenciar carros e pessoas em um carro autnomo ou digitos escritos a mo por pessoas. Como faramos isso? Ora, se tivermos um conjunto de treinamento classificado com esses objetos podemos tratar isso como um aprendizado surpervisionado, certo?

Pegando o Exemplo de Aprendizado Surpervisionado, um dataset muito famoso que tem a ver com esse problema o MNIST dataset.

Revisitando o MNIST Dataset¶

O dataset MNIST contm ambos os dados de treinamento e teste, j devidamente separados. Ele contm 55,000 dados de treinamento e 10,000 dados de teste..

Cada elemento do MNIST tem:

  1. Uma imagem de um digito escrito mo
  2. No caso dos dados de treinamento, a label correspondente (um nmero de 0-9 que identifica a imagem)

Uma forma de resolver esse problema, ento, utilizando tcnicas de aprendizado surpervisionado e uma das mais famosas so as redes neurais !

O que são Redes Neurais Mesmo?¶

Uma rede neural um sistema de "neuronios" artificiais interconectados que trocam mensagens entre si. Essas conexes so ponderadas por pesos que so atualizados a medida que o treinamento ocorre de forma que a rede devidamente treinada seja capaz de responder corretamente quando apresentada uma imagem para reconhecer.

A ideia da ponderao por pesos que eles sejam responsveis por "aprender" partes daquilo que eles devem classificar e para isso, usa-se uma funo de ativao (que ativa os neurnios ou no) e que atualizada por alguma funo de otimizao ao final do processo, geralmente o gradiente descendente.

Mas o que Convolução tem a ver com isso?¶

Ora, se podemos usar pesos combinados com funes de ativao para aprender partes do nosso objetivo, porqu no utilizar essa mesma lgica para aprender as janelas de convoluo? a que Redes Neurais Convolucionais entram ! As CNNs nada mais so do um tipo especial de Rede Neural, cuja arquitetura costuma adicionar algumas camadas a mais alm da tradicional:

- Convolucao: Que faz a mesma coisa que aprendemos anteriormente. A diferena aqui  que os parmetros da janela sero aprendidos conforme ocorrer o ajuste de pesos. Como nas redes tradicionais, aqui tambm aplicamos funes de ativao na saida das redes, para definir se os dados sero usados (ou "ativados").

- Subsampling: Que "Resume" a imagem, reduzindo a resoluo desses features maps, fazendo com que eles sejam melhores contra qualquer tipo de distoro e diminuir o tempo de processamento de imagem. Esse resumo pode ser feito pegando o mximo do vetor ou a mdia entre eles. 

- Fully Connected Layers: Que recebem como entrada esses `features maps` e do como output a imagem escolhida. So as redes neurais tradicionais. 

- Dropout Layer (opcional): Camada usada como regularizao. Ela define uma probabilidade a qual um dado elemento pode ser descartado durante o treinamento.

Entendendo nossos dados:¶

Chamaremos nossas imagens, que sero a entrada da rede, de X e seus labels correspondentes, Y.

As labels sero codificadas como one-hot vectors, que so vetores que possuem 1 na categoria desejada e 0 nos demais.

A Arquitetura da nossa rede ser a seguinte:

  • Convolutional Layer #1: Aplica 32 filtros 5x5 (extraindo 5x5-pixel sub-regies), com a funo de ativao ReLU
  • Pooling Layer #1: Aplica um max pooling com um filtro 2x2 e stride de 2 (que especifica que as regies cujo pooling foi aplicado no se sobrepe)
  • Convolutional Layer #2: Aplica 64 filtros 5x5, com a funo de ativao ReLU
  • Pooling Layer #2: Novamente, aplica max pooling com filtros 2x2 e strides de 2
  • Dense Layer #1: 1,024 neurniocs, com dropout sendo aplicada como regularizador a um rate de 0.4, usando a funnao de ativao reLU
  • Dense Layer #2 (Logits Layer): 10 neurnios, um para cada digito da classe que queremos classificar(09). Aqui, a funo de ativao a ser utilizada a softmax

# Define the neural network
def build_model():
    # This resets all parameters and variables, leave this here
    tf.reset_default_graph()
    
    # Inputs
    net = tflearn.input_data([None, trainX.shape[1]])
    
    net = tf.reshape(net, [-1, 28, 28, 1])

    # Hidden layer(s)
    net = tflearn.conv_2d(net, filter_size=[5,5],nb_filter=32,padding='same', activation=tflearn.activations.relu)
    net= tflearn.max_pool_2d(net, kernel_size=[2,2], strides=2)
    
    net = tflearn.conv_2d(net, filter_size=[5,5],nb_filter=64,padding='same', activation=tflearn.activations.relu)
    net= tflearn.max_pool_2d(net, kernel_size=[2,2], strides=2)
    
    net = tflearn.fully_connected(net, 1024, activation=tflearn.activations.relu)
    net = tflearn.dropout(net, keep_prob = 0.4)

    # Output layer and training model
    
    net = tflearn.fully_connected(net, 10, activation='softmax')
    net = tflearn.regression(net, optimizer='sgd', learning_rate=0.01, loss='categorical_crossentropy')
    
    model = tflearn.DNN(net)
    return model
# Build the model
model = build_model()
